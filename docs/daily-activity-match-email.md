# Daily activity match email (8am local)

This feature runs the activity match **once per day at 8am in each user‚Äôs local timezone** and emails the result from **Ausna Community <community@ausna.co>** when the user has **‚â• 1** matched activity.

## What was added

- **Cron endpoint**: `GET /api/cron/daily-activity-match` (Vercel Cron ‚Üí calls hourly)
- **Daily match computation (service role)**: `lib/explore/dailyExploreMatchService.ts`
- **Email sending (Resend)**:
  - Client + helpers: `lib/email/resend.ts`

## Required environment variables

### Resend

- `RESEND_API_KEY`: Resend API key
- `RESEND_FROM_EMAIL`: must be exactly like:
  - `Ausna Community <community@ausna.co>`

### Supabase (already required for background indexing)

- `SUPABASE_SERVICE_ROLE_KEY`: used by cron + service compute to bypass RLS safely server-side

### Cron security

- `CRON_SECRET`: shared secret
  - Vercel Cron automatically calls your endpoint with header `Authorization: Bearer ${CRON_SECRET}`

### Unsubscribe (optional)

- `UNSUBSCRIBE_SECRET`: secret used to sign one-click unsubscribe tokens (min 16 chars). If not set, `CRON_SECRET` is used.

### URL used in emails

- `NEXT_PUBLIC_SITE_URL`: recommended (e.g. `https://ausna.co`)
  - Fallback: `VERCEL_URL`

## Resend setup: ‚ÄúAusna Community‚Äù

1. In Resend, add and verify the domain **`ausna.co`** (DNS: SPF/DKIM).
2. Create a sender identity:
   - **From name**: `Ausna Community`
   - **From email**: `community@ausna.co`
3. Set `RESEND_FROM_EMAIL="Ausna Community <community@ausna.co>"` in your environment.

## User timezone storage

Each user‚Äôs timezone is read from their **human portfolio** row:

- `portfolios.type = 'human'`
- `metadata.properties.timezone = "<IANA timezone>"` (example: `"Asia/Tokyo"`)

If timezone is missing, that user is skipped by the cron run.

## Vercel Cron setup

This repo includes `vercel.json`:

- Runs hourly: `0 * * * *`
- Calls: `/api/cron/daily-activity-match`

On each invocation, the endpoint checks each user‚Äôs local time and only runs when that user is at **8am** and hasn‚Äôt already run today (based on `metadata.properties.daily_explore_match.ran_at`). Users who have unsubscribed (`metadata.properties.daily_explore_match.unsubscribed === true`) are skipped and do not receive the email.

## Unsubscribe

- **Headers**: Each daily match email includes `List-Unsubscribe` and `List-Unsubscribe-Post` (RFC 8058) so Gmail and other clients can show a one-click ‚ÄúUnsubscribe‚Äù action.
- **Footer**: The email body includes an ‚ÄúUnsubscribe‚Äù link that goes to the same flow.
- **Endpoint**: `GET /api/unsubscribe/daily-match?token=...` ‚Äî verifies the signed token, sets `metadata.properties.daily_explore_match.unsubscribed = true` on the user‚Äôs human portfolio, and shows a confirmation page. `POST` to the same URL is supported for one-click unsubscribe (returns 202).
- **Token**: Generated by `lib/email/unsubscribeToken.ts` (HMAC-SHA256 of user id). No database table is required for tokens.
